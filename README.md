# TubeCutCalculator

TubeCutCalculator — Windows-программа для подготовки расчета лазерного трубореза по CAD-файлам труб. Целевая задача проекта: импорт STEP/STP/IGES/IGS, 3D-просмотр, 2D-развертка, расчет длины реза, количества врезок и стоимости обработки.

Текущая версия: **v0.4.11**.

## Что работает в v0.4.11

- Главное окно PySide6 с drag-and-drop.
- Добавление отдельных файлов, нескольких файлов и папок.
- Рекурсивный поиск CAD-файлов в папке.
- Фильтр форматов `.step`, `.stp`, `.iges`, `.igs`.
- Очередь файлов с пропуском дубликатов.
- Импорт STEP/STP/IGES/IGS через OpenCascade / `pythonocc-core`.
- 3D-просмотр импортированной модели через `OCC.Display.qtDisplay.qtViewer3d`.
- Вертикальное растягивание области 3D/2D-просмотра относительно нижней таблицы файлов.
- Базовый анализ импортированной формы: ось длины, габариты сечения, solid/shell, количество граней и ребер.
- Временное DEV-окно `Geometry Debug Script`: запуск Python-скрипта по текущей модели без пересборки EXE.
- Предварительное определение толщины стенки по граням в толщине изделия.
- Расчет длины реза по наружным границам граней стенки реза; fallback — по 3D B-Rep ребрам `CUT_FEATURE` и `CUT_END`.
- Отдельная длина торцевых резов `CUT_END` и вырезов/пазов `CUT_FEATURE`.
- Расчет количества врезок по связным компонентам `CUT_FEATURE`/`CUT_END`.
- Учет торцов трубы: каждый торцевой наружный контур считается одной врезкой.
- Учет вырезов, которые переходят через несколько наружных граней профильной трубы и скругленные углы.
- Сложный вырез через несколько граней группируется как один cut-face компонент и одна врезка.
- 2D-развертка является только визуализацией классифицированных 3D-ребер и не является источником длины реза.
- Игнорируются внешний контур развертки, рамка, линии сгиба, продольные ребра и линии плоскость/радиус.
- Толщина круглой трубы определяется по `R_outer - R_inner`, если цилиндры найдены надежно.
- Толщина профильной трубы со скруглениями определяется по плоским стенкам наружного/внутреннего контура, без радиусов.
- Ручной ввод толщины для случаев, где автоматическое определение ошибается.
- Confidence толщины и способ определения толщины отображаются в интерфейсе.
- Правая панель параметров прокручивается и не увеличивает стартовое окно выше экрана.
- Диагностический режим `debug_edges.csv`: каждое 3D B-Rep ребро, длина, тип, входит ли оно в расчет и причина.
- В таблице результата отдельно показаны реальная длина реза, торцевые резы, вырезы/пазы, диагностическая сумма всех CAD-ребер, врезки, игнорированные продольные ребра и линии плоскость/радиус.
- Резервный расчет по площади граней толщины, если наружный контур не найден.
- Fallback-отбор реза по границам наружная поверхность/грань выреза и wire-петлям граней.
- Верхнее меню: `Файл`, `Настройки`, `Печать / экспорт`.
- Светлая и темная тема, настройка сохраняется между запусками.
- Упрощенная расчетная таблица: файл, размер/толщина, длина, длина реза, врезки, цена.
- Отдельная диагностическая вкладка для технических колонок и проверки расчета.
- Настройки контрагентов с индивидуальной наценкой и валютой.
- Настройки материалов, включая стандартную длину хлыста и ориентировочную цену трубы.
- Таблица цен по контрагенту, материалу и толщине.
- Подбор цены по точному правилу, ближайшей толщине в допуске или цене по умолчанию с предупреждением.
- Расчет стоимости с общей наценкой и наценкой контрагента.
- Расчет закупки трубы с группировкой по материалу, типу профиля, размеру и толщине.
- Вкладка `Закупка трубы`: количество деталей, длина, припуски, запас, хлысты, закупка и остаток.
- Форма данных коммерческого предложения.
- Загрузка логотипа PNG/JPG для отчетов.
- Экспорт CSV.
- Экспорт Excel `.xlsx` с листами `Расчет деталей`, `Закупка трубы`, `Настройки`.
- PDF коммерческого предложения.
- PDF технического отчета.
- Печать текущего расчета, коммерческого предложения и технического отчета через системный диалог.
- Предпросмотр печати.
- Сохранение и открытие проекта JSON со списком деталей, расчетами и настройками.
- GitHub-ready структура проекта.
- `build_exe.bat` для сборки Windows EXE через conda + PyInstaller.
- GitHub Actions workflow для сборки EXE на Windows runner.

2D-развертка в v0.4.11 — диагностическая карта расчета, а не готовая производственная развертка/DXF. Источником расчета длины реза является 3D B-Rep анализ граней стенки реза; классификация отдельных ребер используется как fallback.

## Установка зависимостей

Для v0.4.11 рекомендуется Miniforge или Anaconda, потому что `pythonocc-core` устанавливается из `conda-forge`.

```bat
conda env create -f environment.yml
conda activate TubeCutCalculator
```

Если окружение уже создано:

```bat
conda env update -n TubeCutCalculator -f environment.yml --prune
conda activate TubeCutCalculator
```

## Запуск из исходников

```bat
conda activate TubeCutCalculator
python main.py
```

## Сборка EXE

На Windows запустите:

```bat
build_exe.bat
```

Скрипт создаст или обновит conda-окружение `TubeCutCalculator`, затем соберет one-file EXE через `TubeCutCalculator.spec`.

Результат:

- `dist/TubeCutCalculator.exe`
- `dist/version.txt`

## GitHub Actions

Workflow `.github/workflows/build-windows.yml` собирает EXE на Windows runner при push в `main`, push в `feature/**` и при ручном запуске. После сборки workflow запускает smoke-test импортов внутри готового EXE. При создании тега вида `v0.4.11` EXE также прикладывается к GitHub Release.

```bat
git tag v0.4.11
git push origin v0.4.11
```
