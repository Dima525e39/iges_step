# TubeCutCalculator

TubeCutCalculator — Windows-программа для расчета лазерной резки труб и листовых DXF-деталей. Программа импортирует CAD-файлы, показывает 3D/2D-предпросмотр, считает длину реза, количество врезок, стоимость обработки, закупку трубы и формирует Excel/PDF-отчеты.

Текущая версия: **v0.5.5**.

## Главное

- Импорт STEP / STP / IGES / IGS через OpenCascade (`pythonocc-core`).
- Импорт DXF для листовых 2D-деталей.
- Drag-and-drop файлов и папок.
- 3D-просмотр труб и 2D-просмотр контуров.
- Расчет длины реза и врезок по 3D B-Rep ребрам трубы.
- Отдельный fallback-анализ круглых IGES-труб, записанных BSpline-поверхностями.
- Ручной ввод толщины, если автоопределение ошибается.
- Таблица расчета: файл, материал, размер, толщина, длина, длина реза, врезки, количество, цена.
- Пересчет стоимости при изменении материала, толщины или количества.
- Настройки контрагентов, материалов, цен и закупки трубы.
- Расчет закупки трубы с учетом полезной длины хлыста и остатка в патроне станка.
- Excel-экспорт, PDF-счет/КП, PDF-технический отчет и печать.
- Сохранение и открытие проекта JSON.
- Диагностика `debug_edges.csv` и `debug_faces.csv`.
- Windows EXE-сборка через `build_exe.bat`.
- GitHub Actions сборка EXE-артефакта.

## Поддерживаемые форматы

- `.step`
- `.stp`
- `.iges`
- `.igs`
- `.dxf`

DXF сейчас используется для листовых деталей. STEP/IGES используются для труб и 3D-моделей.

## Логика расчета труб

Для труб расчет длины реза не берется из 2D-развертки. Источник расчета — 3D B-Rep модель.

Основные правила:

- В длину реза входят только ребра, классифицированные как `CUT_END` и `CUT_FEATURE`.
- `CUT_END` — торцевой рез трубы.
- `CUT_FEATURE` — отверстие, паз или другой вырез на трубе.
- Продольные ребра трубы не считаются резом.
- Линии профиля, линии плоскость/радиус и вспомогательные линии развертки не считаются резом.
- Внешняя рамка 2D-развертки не является резом.
- Количество врезок считается по связным контурам `CUT_END` и `CUT_FEATURE`, а не по количеству отдельных ребер.
- Каждый торец трубы считается отдельной врезкой.

Для профильных труб программа пытается находить наружные грани, грани стенки реза и связанные контуры вырезов. Вырезы, переходящие через несколько граней и скругленные углы, должны группироваться как один контур.

Для круглых труб есть отдельные ветки анализа:

- расчет по наружным цилиндрическим граням, если OpenCascade надежно определил цилиндр;
- fallback по bbox круглой BSpline-трубы, если IGES записан набором BSpline-поверхностей;
- fallback по поперечным B-Rep ребрам, если наружный цилиндр не найден.

Толщина круглой трубы определяется как `R_outer - R_inner`, когда наружный и внутренний контур найдены надежно. Для профильных труб толщина определяется по плоским стенкам, без радиусов.

## Диагностика

В правой панели можно включить `debug_edges.csv`.

При включенной диагностике программа сохраняет:

- `debug_edges.csv` — каждое B-Rep ребро, длина, тип, входит ли оно в расчет, причина классификации, wire-роли и bbox.
- `debug_faces.csv` — грани модели, тип поверхности, радиус, bbox и причина классификации.

В CSV также записываются:

- `app_version`
- `build_commit`
- `calc_core`

Для актуального EXE `build_commit` должен быть реальным SHA коммита. Если там `unknown`, значит запущен неподтвержденный или неправильно собранный EXE.

## Листовые DXF и nesting

Для DXF-листов расчет идет по 2D-контурам DXF:

- внешний контур листовой детали входит в рез;
- внутренние контуры считаются отдельными врезками;
- сегменты `LINE`, `ARC`, `SPLINE`, старые `POLYLINE`/`VERTEX`/`SEQEND` собираются в контуры;
- количество деталей может парситься из имени файла;
- окно `Nesting` поддерживает предпросмотр раскладки, зазор, несколько листов и поворот деталей на произвольный угол с настраиваемым шагом.

Nesting пока является рабочим расчетным модулем для проверки DXF-раскладки. Производственная оптимизация большого количества сложных контуров остается отдельным этапом развития.

## Установка зависимостей

Рекомендуется Miniforge или Anaconda, потому что `pythonocc-core` устанавливается из `conda-forge`.

```bat
conda env create -f environment.yml
conda activate TubeCutCalculator
```

Если окружение уже создано:

```bat
conda env update -n TubeCutCalculator -f environment.yml --prune
conda activate TubeCutCalculator
```

## Запуск из исходников

```bat
conda activate TubeCutCalculator
python main.py
```

## Сборка EXE на Windows

```bat
build_exe.bat
```

Скрипт:

- создает или обновляет conda-окружение `TubeCutCalculator`;
- записывает `app_build.py` с текущим commit/date/core;
- собирает one-file EXE через PyInstaller;
- копирует `version.txt` в `dist/`.

Результат:

```text
dist/TubeCutCalculator.exe
dist/version.txt
```

## GitHub Actions

Workflow `.github/workflows/build-windows.yml` собирает EXE на Windows runner при:

- push в `main`;
- push в `feature/**`;
- ручном запуске `workflow_dispatch`;
- push тега вида `v*`.

После сборки workflow запускает packaged self-test:

```bat
TubeCutCalculator.exe --self-test-imports
```

Если self-test не проходит, artifact не публикуется. При создании тега EXE-архив также прикладывается к GitHub Release.

Пример релиза:

```bat
git tag v0.5.5
git push origin v0.5.5
```

## Ограничения

- Расчет трубы зависит от качества CAD-топологии: плохие IGES-файлы могут требовать fallback или ручной толщины.
- 2D-развертка трубы является диагностической визуализацией, а не источником расчета.
- Nesting для DXF развивается как практический true-shape модуль, но еще не является финальным промышленным оптимизатором для всех случаев.
